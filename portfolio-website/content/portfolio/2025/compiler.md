## Overview

This project was completed in **BYU CS 236: Discrete Structures**. Over the course of the class, I built a Datalog scanner, parser, relational database, interpreter, and optimized rule evaluation pipeline in C++.

More information about the project sequence is available on the [course lab page](https://faculty.cs.byu.edu/~barker/cs236/labs/labs.php).

## Project Components

**[Data Scanner](https://faculty.cs.byu.edu/~barker/cs236/labs/lab1-extra.html)**: Read a text file, identify Datalog tokens, and output the resulting token stream.

**[Data Parser](https://faculty.cs.byu.edu/~barker/cs236/labs/lab2-extra.html)**: Parse a Datalog program and build a structured representation of its contents.

**[The Relational Database](https://faculty.cs.byu.edu/~barker/cs236/labs/lab3-extra.html)**: Use relational operations to evaluate queries from Datalog facts.

**[The Datalog Interpreter](https://faculty.cs.byu.edu/~barker/cs236/labs/lab4-extra.html)**: Add join and union operations to evaluate Datalog rules.

**[Optimizing Rule Evaluation](https://faculty.cs.byu.edu/~barker/cs236/labs/lab5-extra.html)**: Build a dependency graph and apply depth-first search to improve rule evaluation performance.
